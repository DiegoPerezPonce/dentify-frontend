- Route Guards by Roles (RBAC) #4
- Description: Restrict access using exactly two Symfony-style roles: ROLE_ADMIN and ROLE_USER. Examples: routes/modules for user CRUD, role assignment (ADMIN vs USER only), boxes, treatment protocols, and stock configuration are ROLE_ADMIN only; patient care flows (dashboard, patients, appointments, odontogram, radiographs, clinical history, schedules as consumed by the app) require an authenticated user with ROLE_USER (admins typically hold ROLE_ADMIN and may also have ROLE_USER for day-to-day screens—define in JWT/backend). Only the login route stays public (no JWT). There is no public application signup: new user accounts are created exclusively by ROLE_ADMIN via issue #15 (and backend admin APIs). Implement route guards and optional menu visibility from JWT claims.
- Branch: feature/4-route-guards-by-roles-rbac
